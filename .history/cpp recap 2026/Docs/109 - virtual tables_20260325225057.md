# 109 - Virtual Tables (vtable) in C++

## What is a Virtual Table?
A virtual table (vtable) is a mechanism used by C++ to implement dynamic dispatch (runtime polymorphism). It's an array of function pointers that stores addresses of virtual functions for a class. Each class with virtual functions has its own vtable, and each object contains a hidden pointer (vptr) to its class's vtable.

## Type 1: Single File Implementation

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void func1() {
        cout << "Base::func1" << endl;
    }
    
    virtual void func2() {
        cout << "Base::func2" << endl;
    }
    
    virtual void func3() {
        cout << "Base::func3" << endl;
    }
    
    void nonVirtual() {
        cout << "Base::nonVirtual" << endl;
    }
    
    virtual ~Base() {}
};

class Derived : public Base {
public:
    void func1() override {
        cout << "Derived::func1" << endl;
    }
    
    void func2() override {
        cout << "Derived::func2" << endl;
    }
    
    void func4() {
        cout << "Derived::func4" << endl;
    }
};

int main() {
    Base* ptr = new Derived();
    
    // Virtual function calls - resolved through vtable
    ptr->func1();  // Calls Derived::func1
    ptr->func2();  // Calls Derived::func2
    ptr->func3();  // Calls Base::func3 (not overridden)
    
    // Non-virtual function - resolved at compile time
    ptr->nonVirtual();  // Calls Base::nonVirtual
    
    delete ptr;
    
    return 0;
}
```

## Understanding vtable Layout

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void f() { cout << "Base::f" << endl; }
    virtual void g() { cout << "Base::g" << endl; }
    virtual void h() { cout << "Base::h" << endl; }
    virtual ~Base() {}
};

class Derived1 : public Base {
public:
    void f() override { cout << "Derived1::f" << endl; }
    void h() override { cout << "Derived1::h" << endl; }
};

class Derived2 : public Base {
public:
    void g() override { cout << "Derived2::g" << endl; }
};

int main() {
    cout << "=== Virtual Table Demonstration ===" << endl;
    
    Base* b1 = new Base();
    Base* d1 = new Derived1();
    Base* d2 = new Derived2();
    
    cout << "Base pointer to Base object:" << endl;
    b1->f();  // Base::f
    b1->g();  // Base::g
    b1->h();  // Base::h
    
    cout << "\nBase pointer to Derived1 object:" << endl;
    d1->f();  // Derived1::f (overridden)
    d1->g();  // Base::g (not overridden)
    d1->h();  // Derived1::h (overridden)
    
    cout << "\nBase pointer to Derived2 object:" << endl;
    d2->f();  // Base::f (not overridden)
    d2->g();  // Derived2::g (overridden)
    d2->h();  // Base::h (not overridden)
    
    delete b1;
    delete d1;
    delete d2;
    
    return 0;
}
```

## Visualizing vtable Structure

```cpp
#include <iostream>
#include <cstdint>
using namespace std;

class Simple {
public:
    virtual void a() { cout << "Simple::a" << endl; }
    virtual void b() { cout << "Simple::b" << endl; }
    virtual ~Simple() {}
};

class Extended : public Simple {
public:
    void a() override { cout << "Extended::a" << endl; }
    virtual void c() { cout << "Extended::c" << endl; }
};

int main() {
    Simple s;
    Extended e;
    
    cout << "Size of Simple: " << sizeof(Simple) << " bytes" << endl;
    cout << "Size of Extended: " << sizeof(Extended) << " bytes" << endl;
    
    // The extra bytes are for the vptr (virtual pointer)
    // On 64-bit systems, vptr is 8 bytes
    
    Simple* ptr = &e;
    ptr->a();  // Extended::a (through vtable)
    ptr->b();  // Simple::b
    // ptr->c();  // Not accessible through Simple*
    
    return 0;
}
```

## vtable Performance Impact

```cpp
#include <iostream>
#include <chrono>
using namespace std;

class VirtualClass {
public:
    virtual int compute(int x) {
        return x * x;
    }
    virtual ~VirtualClass() {}
};

class NonVirtualClass {
public:
    int compute(int x) {
        return x * x;
    }
};

class OverrideClass : public VirtualClass {
public:
    int compute(int x) override {
        return x * x * x;
    }
};

int main() {
    const int ITERATIONS = 100000000;
    
    // Non-virtual call
    NonVirtualClass nonVirtual;
    auto start = chrono::high_resolution_clock::now();
    int sum = 0;
    for (int i = 0; i < ITERATIONS; i++) {
        sum += nonVirtual.compute(i);
    }
    auto end = chrono::high_resolution_clock::now();
    auto nonVirtualTime = chrono::duration_cast<chrono::milliseconds>(end - start);
    
    // Virtual call (base pointer to base object)
    VirtualClass* vptr = new VirtualClass();
    start = chrono::high_resolution_clock::now();
    sum = 0;
    for (int i = 0; i < ITERATIONS; i++) {
        sum += vptr->compute(i);
    }
    end = chrono::high_resolution_clock::now();
    auto virtualTime = chrono::duration_cast<chrono::milliseconds>(end - start);
    
    // Virtual call (base pointer to derived object)
    delete vptr;
    vptr = new OverrideClass();
    start = chrono::high_resolution_clock::now();
    sum = 0;
    for (int i = 0; i < ITERATIONS; i++) {
        sum += vptr->compute(i);
    }
    end = chrono::high_resolution_clock::now();
    auto overrideTime = chrono::duration_cast<chrono::milliseconds>(end - start);
    
    delete vptr;
    
    cout << "Non-virtual call time: " << nonVirtualTime.count() << " ms" << endl;
    cout << "Virtual call (base): " << virtualTime.count() << " ms" << endl;
    cout << "Virtual call (override): " << overrideTime.count() << " ms" << endl;
    cout << "Virtual call overhead: " 
         << (virtualTime.count() - nonVirtualTime.count()) << " ms" << endl;
    
    return 0;
}
```

## vtable and Memory Layout

```cpp
#include <iostream>
#include <cstdint>
using namespace std;

class BaseWithVTable {
public:
    int baseData;
    virtual void func() { cout << "Base::func" << endl; }
    virtual ~BaseWithVTable() {}
};

class DerivedWithVTable : public BaseWithVTable {
public:
    int derivedData;
    void func() override { cout << "Derived::func" << endl; }
};

int main() {
    BaseWithVTable b;
    DerivedWithVTable d;
    
    cout << "=== Memory Layout ===" << endl;
    cout << "Size of BaseWithVTable: " << sizeof(BaseWithVTable) << " bytes" << endl;
    cout << "Size of DerivedWithVTable: " << sizeof(DerivedWithVTable) << " bytes" << endl;
    
    // Print memory addresses
    cout << "\nBase object addresses:" << endl;
    cout << "&b: " << &b << endl;
    cout << "&b.baseData: " << &b.baseData << endl;
    
    cout << "\nDerived object addresses:" << endl;
    cout << "&d: " << &d << endl;
    cout << "&d.baseData: " << &d.baseData << endl;
    cout << "&d.derivedData: " << &d.derivedData << endl;
    
    // The first 8 bytes (on 64-bit) are the vptr
    cout << "\nOffset to baseData: "
```

## vtable in Diamond Inheritance

```cpp

using namespace std;

class Root {

    virtual void rootFunc() { cout << "Root::rootFunc" << endl; }
    virtual ~Root() {}

public:
    virtual void branch1Func() { cout << "Branch1::branch1Func" << endl; }
};

class Branch2 : public virtual Root {
public:
    virtual void branch2Func() { cout << "Branch2::branch2Func" << endl; }
};

class Leaf : public Branch1, public Branch2 {
public:
    void rootFunc() override { cout << "Leaf::rootFunc" << endl; }
    virtual void leafFunc() { cout << "Leaf::leafFunc" << endl; }
};

int main() {
    cout << "Size of Root: " << sizeof(Root) << " bytes" << endl;
    cout << "Size of Branch1: " << sizeof(Branch1) << " bytes" << endl;

    Branch1* b1 = &leaf;

    b2->branch2Func();  // Branch2::branch2Func
```

## Common vtable Misconceptions

```cpp
#include <iostream>
using namespace std;

class A {
public:
    virtual void f() { cout << "A::f" << endl; }
    virtual ~A() {}
};

class B : public A {
public:
    void f() override { cout << "B::f" << endl; }
};

class C : public B {
public:
    void f() override { cout << "C::f" << endl; }
};

int main() {
    // Misconception 1: vtable is per object, not per class
    // Truth: vtable is per class, objects share the same vtable
    
    A a1, a2;
    B b1, b2;
    
    cout << "vtable address for a1: " << *(void**)&a1 << endl;
    cout << "vtable address for a2: " << *(void**)&a2 << endl;
    cout << "vtable address for b1: " << *(void**)&b1 << endl;
    cout << "vtable address for b2: " << *(void**)&b2 << endl;
    
    // Misconception 2: Virtual functions are slower always
    // Truth: Only one extra indirection (vptr lookup)
    
    // Misconception 3: Virtual functions cannot be inlined
    // Truth: Compiler can inline when object type is known at compile time
    
    A* ptr = new C();
    ptr->f();  // C::f (runtime dispatch)
    
    C c;
    c.f();  // C::f (can be inlined by compiler)
    
    delete ptr;
    
    return 0;
}
```

## vtable Implementation Details

```cpp
#include <iostream>
using namespace std;

// Simulating vtable manually
class ManualVTable {
public:
    typedef void (*FunctionPtr)();
    
    struct VTable {
        FunctionPtr func1;
        FunctionPtr func2;
        FunctionPtr destructor;
    };
    
    static void base_func1() { cout << "Base::func1" << endl; }
    static void base_func2() { cout << "Base::func2" << endl; }
    static void base_destructor() { cout << "Base destructor" << endl; }
    
    static void derived_func1() { cout << "Derived::func1" << endl; }
    static void derived_destructor() { cout << "Derived destructor" << endl; }
    
    static VTable base_vtable;
    static VTable derived_vtable;
    
    VTable* vptr;
    
    ManualVTable(bool isDerived) {
        vptr = isDerived ? &derived_vtable : &base_vtable;
    }
    
    void callFunc1() { vptr->func1(); }
    void callFunc2() { vptr->func2(); }
    void destroy() { vptr->destructor(); }
};

ManualVTable::VTable ManualVTable::base_vtable = {
    base_func1,
    base_func2,
    base_destructor
};

ManualVTable::VTable ManualVTable::derived_vtable = {
    derived_func1,
    base_func2,
    derived_destructor
};

int main() {
    ManualVTable base(false);
    ManualVTable derived(true);
    
    base.callFunc1();   // Base::func1
    base.callFunc2();   // Base::func2
    
    derived.callFunc1(); // Derived::func1
    derived.callFunc2(); // Base::func2
    
    derived.destroy();   // Derived destructor
    
    return 0;
}
```

## vtable Summary Table

| Aspect | Description |
|--------|-------------|
| **vptr** | Hidden pointer in each object pointing to vtable |
| **vtable** | Array of function pointers for virtual functions |
| **Size** | One vptr per object (8 bytes on 64-bit) |
| **Creation** | Compiler generates vtable for each class with virtual functions |
| **Inheritance** | Derived classes get copy of base vtable with overrides |
| **Multiple Inheritance** | Multiple vtables (one per base class) |
| **Virtual Inheritance** | More complex vtable structure with offsets |
| **Performance** | One extra indirection per virtual call |

## Common Misconceptions

| Misconception | Truth |
|---------------|-------|
| vtable is stored per object | vtable is per class, vptr is per object |
| Virtual functions are always slow | Only one extra indirection overhead |
| Virtual functions cannot be inlined | Can be inlined when type is known |
| All classes have vtables | Only classes with virtual functions |
| vtables cause memory bloat | Only one vtable per class, not per object |

## Summary

- **vtable** is an array of function pointers for virtual functions
- **vptr** is a hidden pointer in each object pointing to the vtable
- Enables **runtime polymorphism** through dynamic dispatch
- Created by compiler for classes with virtual functions
- **Single inheritance**: one vtable, vptr points to it
- **Multiple inheritance**: multiple vtables, multiple vptrs
- **Virtual inheritance**: more complex vtable with offset information
- **Performance**: small overhead (one indirection per virtual call)
- **Memory**: one vptr per object (8 bytes on 64-bit systems)
