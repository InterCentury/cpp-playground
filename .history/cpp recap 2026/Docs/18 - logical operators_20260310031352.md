# 18 - Logical Operators in C++

## What are Logical Operators?
Logical operators combine multiple conditions and return a boolean result (`true` or `false`). They are used to make complex decisions in programs.

## The Three Logical Operators

| Operator | Name | Example | Result |
|----------|------|---------|--------|
| `&&` | AND | `(5 > 3) && (2 < 4)` | `true` |
| `||` | OR | `(5 < 3) || (2 < 4)` | `true` |
| `!` | NOT | `!(5 < 3)` | `true` |

## Simple Examples

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << boolalpha;
    
    // AND (&&) - both must be true
    cout << "true && true: " << (true && true) << endl;     // true
    cout << "true && false: " << (true && false) << endl;   // false
    cout << "false && true: " << (false && true) << endl;   // false
    cout << "false && false: " << (false && false) << endl; // false
    
    cout << "\n";
    
    // OR (||) - at least one must be true
    cout << "true || true: " << (true || true) << endl;     // true
    cout << "true || false: " << (true || false) << endl;   // true
    cout << "false || true: " << (false || true) << endl;   // true
    cout << "false || false: " << (false || false) << endl; // false
    
    cout << "\n";
    
    // NOT (!) - reverses the value
    cout << "!true: " << (!true) << endl;     // false
    cout << "!false: " << (!false) << endl;   // true
    
    return 0;
}
```

## Logical AND (&&)

```cpp
#include <iostream>
using namespace std;

int main() {
    int age = 25;
    bool hasLicense = true;
    
    cout << boolalpha;
    
    // Both conditions must be true
    bool canDrive = (age >= 18) && (hasLicense == true);
    
    cout << "Age: " << age << endl;
    cout << "Has license: " << hasLicense << endl;
    cout << "Can drive: " << canDrive << endl;  // true
    
    // Change one condition
    age = 16;
    canDrive = (age >= 18) && (hasLicense == true);
    cout << "\nAge: " << age << endl;
    cout << "Can drive: " << canDrive << endl;  // false
    
    return 0;
}
```

## Logical OR (||)

```cpp
#include <iostream>
using namespace std;

int main() {
    bool isWeekend = true;
    bool isHoliday = false;
    
    cout << boolalpha;
    
    // At least one condition must be true
    bool canSleepIn = isWeekend || isHoliday;
    
    cout << "Weekend: " << isWeekend << endl;
    cout << "Holiday: " << isHoliday << endl;
    cout << "Can sleep in: " << canSleepIn << endl;  // true
    
    // Both false
    isWeekend = false;
    isHoliday = false;
    canSleepIn = isWeekend || isHoliday;
    cout << "\nWeekend: " << isWeekend << endl;
    cout << "Holiday: " << isHoliday << endl;
    cout << "Can sleep in: " << canSleepIn << endl;  // false
    
    return 0;
}
```

## Logical NOT (!)

```cpp
#include <iostream>
using namespace std;

int main() {
    bool isRaining = true;
    bool isSunny = false;
    
    cout << boolalpha;
    
    cout << "Is raining: " << isRaining << endl;
    cout << "Not raining: " << (!isRaining) << endl;  // false
    
    cout << "\nIs sunny: " << isSunny << endl;
    cout << "Not sunny: " << (!isSunny) << endl;      // true
    
    // Double NOT returns original
    cout << "\n!!true: " << (!!true) << endl;        // true
    cout << "!!false: " << (!!false) << endl;        // false
    
    return 0;
}
```

## Truth Tables

### AND (&&) Truth Table
| A | B | A && B |
|---|---|--------|
| false | false | false |
| false | true | false |
| true | false | false |
| true | true | true |

### OR (||) Truth Table
| A | B | A \|\| B |
|---|---|--------|
| false | false | false |
| false | true | true |
| true | false | true |
| true | true | true |

### NOT (!) Truth Table
| A | !A |
|---|---|
| true | false |
| false | true |

## Combining Multiple Operators

```cpp
#include <iostream>
using namespace std;

int main() {
    int x = 10, y = 5, z = 8;
    
    cout << boolalpha;
    
    // Complex conditions
    bool result1 = (x > y) && (y < z) || (z == 8);
    // (true && true) || true = true || true = true
    
    bool result2 = (x < y) || (y > z) && (z == 5);
    // false || false && false = false || false = false
    
    // Using parentheses for clarity
    bool result3 = ((x > y) && (y < z)) || (z == 8);
    
    cout << "Result 1: " << result1 << endl;
    cout << "Result 2: " << result2 << endl;
    cout << "Result 3: " << result3 << endl;
    
    return 0;
}
```

## Short-Circuit Evaluation

C++ stops evaluating as soon as the result is determined.

```cpp
#include <iostream>
using namespace std;

int main() {
    int x = 5;
    int y = 0;
    
    // AND short-circuit: if first is false, second not evaluated
    if (y != 0 && (x / y > 2)) {
        cout << "This won't cause division by zero!" << endl;
    } else {
        cout << "Short-circuit prevented division by zero" << endl;
    }
    
    // OR short-circuit: if first is true, second not evaluated
    int count = 0;
    if (count > 0 || (count = 10) > 0) {
        cout << "Count is now: " << count << endl;  // count still 0
    }
    
    return 0;
}
```

## De Morgan's Laws

```cpp
#include <iostream>
using namespace std;

int main() {
    bool a = true, b = false;
    
    cout << boolalpha;
    
    // !(a && b) is same as (!a) || (!b)
    cout << "!(true && false) = " << !(a && b) << endl;      // true
    cout << "!true || !false = " << (!a || !b) << endl;      // true
    
    // !(a || b) is same as (!a) && (!b)
    cout << "\n!(true || false) = " << !(a || b) << endl;    // false
    cout << "!true && !false = " << (!a && !b) << endl;      // false
    
    return 0;
}
```

## Common Mistakes

### Mistake 1: Using & or | Instead of && or ||
```cpp
int x = 5;

// Wrong - bitwise operators, not logical
if (x > 3 & x < 10) { }  // Works but not correct

// Correct
if (x > 3 && x < 10) { }
```

### Mistake 2: Forgetting Parentheses
```cpp
int a = 5, b = 10, c = 15;

// Wrong - precedence issue
if (a > b && b > c || c > a) { }  // Unclear

// Better
if ((a > b && b > c) || (c > a)) { }
```

### Mistake 3: Redundant Comparisons
```cpp
bool flag = true;

// Wrong - redundant
if (flag == true) { }

// Correct
if (flag) { }

// Wrong
if (flag == false) { }

// Correct
if (!flag) { }
```

### Mistake 4: Range Checking
```cpp
int x = 5;

// Wrong - doesn't work as expected
if (1 < x < 10) { }  // Always true!

// Correct
if (x > 1 && x < 10) { }
```

### Mistake 5: Assignment in Condition
```cpp
int x = 5;

// Wrong - assignment, not comparison
if (x = 10) { }  // Always true, x becomes 10

// Correct
if (x == 10) { }
```

## Operator Precedence

```cpp
#include <iostream>
using namespace std;

int main() {
    // Precedence: ! > && > ||
    
    bool result1 = true || false && false;
    // true || (false && false) = true || false = true
    
    bool result2 = (true || false) && false;
    // true && false = false
    
    bool result3 = !true && false;
    // (!true) && false = false && false = false
    
    cout << boolalpha;
    cout << "true || false && false: " << result1 << endl;
    cout << "(true || false) && false: " << result2 << endl;
    cout << "!true && false: " << result3 << endl;
    
    return 0;
}
```

## Summary

- **&& (AND)** - true only if both conditions are true
- **|| (OR)** - true if at least one condition is true
- **! (NOT)** - reverses the boolean value
- Use for complex conditions and decision making
- Short-circuit evaluation prevents unnecessary calculations
- De Morgan's laws help simplify negated conditions
- Parentheses make complex conditions clearer
- Avoid redundant comparisons with boolean variables
- Remember operator precedence: `!` > `&&` > `||`
- Never use `=` when you mean `==`
